# Jobs-to-be-Done
 
Jobs-to-be-Done is a framework that frames a product or feature around the underlying task or goal a user is trying to accomplish, rather than around the user's demographic traits. It asks what job the user is hiring a product to do, which can reveal competitors that are not obvious, such as a notepad app competing with a to-do list app for the same job. This framing helps teams avoid designing around surface-level assumptions about who a user is.
