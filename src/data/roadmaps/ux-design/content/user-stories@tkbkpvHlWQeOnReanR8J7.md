# User Stories

A user story is a short description of a feature written from the perspective of the user, typically following the format "as a \[user\], I want \[goal\] so that \[benefit\]." It keeps the product backlog focused on user needs rather than technical implementation details. Teams use user stories to prioritize what to build and to write acceptance criteria for each feature.
